## レポート（日報・点検記録）を効率的に探すための方法{#findReportMethod}

日報や点検表、業務報告書は、基本的に毎日作成され、時間とともに膨大な情報が蓄積されていきます。そのため、**効率よく目的のレポートを探し出す仕組み**は、業務の生産性向上に不可欠です。

一般的なチャット日報の場合、情報はスレッドにどんどん流れていくため、後から特定の日の日報や、特定の話題に関する報告を探し出すのは非常に困難です。しかしNipoPlusでは、こういった**情報の埋没や見落としが起きないよう**、多様な検索・閲覧機能が備わっています。

未読のレポートだけを表示したり、カレンダーから直感的に探したり、詳細な検索条件で絞り込んだりといった様々な方法で、**目的のレポートを素早く見つけ出し、必要な情報を活用**しましょう。

<dl class="basic">
<dt><a href="#calendar"><strong>カレンダーからレポートを探す</strong></a></dt>
<dd>月間カレンダーや週カレンダー上にレポートや予定が直感的に表示されます。<strong>特定の日付の記録</strong>を素早く確認したい場合に便利です。</dd>
<dt><a href="#listbox"><strong>保存箱（受信BOX・送信BOX・下書きBOX）からレポートを探す</strong></a></dt>
<dd>期間指定やフィルタ、キーワード検索など、<strong>詳細な条件で目的のレポートを絞り込める</strong>強力な機能です。NipoPlusの主要な情報探索場所となります。</dd>
<dt><a href="/docs/manual/utils/notice/"><strong>通知から直接レポートを開く</strong></a></dt>
<dd>あなた宛にレポートが作成されると通知が発行されます。この通知をクリックすることで直接そのレポートを閲覧できます。<strong>メール通知</strong>をONにしている場合はメールからも直接レポートを開けます。</dd>
</dl>

## カレンダーを使ってレポート（日報・点検記録）を探す{#calendar}

{{< callout context="note" title="カレンダーがメニューに無い場合は機能の有効化が必要です" icon="outline/info-circle" >}}
カレンダー機能が左メニューに表示されない場合は、[グループ設定](/docs/setup/setting-group/#optionalFunction)から機能を有効にしてください。
{{< /callout >}}

提出された<strong>日報や点検記録</strong>はカレンダー上に掲載されます。日別や週別でレポートを読みたい場合は、カレンダー機能が非常に便利です。

1.  メニュー「<strong>カレンダー</strong>」を開く
2.  右パネル上の方にある「受・送」が「<strong>受</strong>」（受信したレポート）になっていることを確認する
3.  右パネルの「レポートを表示」がONであることを確認する
4.  カレンダー上の「<strong>〇件のレポート</strong>」という文字をクリックする

{{<icatch filename="img/read-report-calendar" msg="カレンダー画面です。日報や点検記録、予定が1枚のカレンダー上にまとめて表示されます。特定の日付の記録を視覚的に把握できます" alice="guide">}}

「〇件のレポート」をクリックすると、ポップアップで該当するレポートが表示されます。「〇件」の数だけページ送りができます。
カレンダー機能についてより詳しいガイドは[カレンダー](/docs/manual/calendar/_about/)を御覧ください。
レポートを表示後の操作については[レポートを読む](/docs/manual/read-report/state/)を御覧ください。

## 保存箱を使ってレポート（日報・点検記録）を探す{#listbox}

「保存箱」は、あなたが書いたレポート、受信したレポート、下書きのレポートが格納される中心的な場所です。NipoPlusの様々な検索・絞り込み機能を活用し、効率的に目的のレポートを見つけましょう。

{{< callout context="caution" title="権限によって表示される項目が異なります" icon="outline/alert-triangle" >}}
[権限](/docs/setup/staff-global/rank/)が「ゲスト」のアカウントは「保存箱」自体表示されません。「書き込みのみ」のアカウントは「受信BOX」が表示されません。
{{< /callout >}}

保存箱の外観は次のように見えます。

{{<icatch filename="img/report-box1" msg="提出された日報や点検記録は保存箱からアクセスできます。あなたの権限に応じて表示されるBOXが異なります" alice="ok">}}

基本的な使い方はシンプルです。

1.  右パネルのカレンダーから表示したい期間を選択する
2.  表示された一覧から任意のレポートをクリック

これだけです。

プレイボタン（赤枠に▶アイコン）をクリックすると「ページを切り替えてレポートを表示」します。それ以外の場所をクリックすると「ポップアップでレポートを表示」します。
見た目が違いますが表示内容は同じです。

### 検索機能を使いこなす - 目的のレポートを素早く絞り込む{#searchFunction}

受信BOXは「高速モード」と「検索モード」があります。高速モードはその名の通り、期間内のレポートを一瞬で表示しますが、細かい検索はできません。検索や並べ替えを使うと自動で「検索モード」に切り替わります。

1.  <strong>並べ替え</strong>：並べ替えしたい列のヘッダーをクリックします。1度のクリックで昇順、もう一度クリックで降順となります。提出日時、作成者、テンプレート名などで並べ替えることができます。
2.  <strong>絞り込み</strong>：提出者や[タグ](/docs/setup/advanced-setting/tag/)、テンプレート、キーワードなどで絞り込みたい場合、それぞれの列で絞り込む対象を選択します。

様々な条件を組み合わせて指定できます。例えば次の画像は「検索期間が今月で提出日の昇順」「提出者が『〇〇部』」「レポートに『重要』タグがセットされている」という条件で検索した例です。**特定の設備に関する点検記録**や、**特定のプロジェクトの日報**などを素早く探し出すのに役立ちます。

{{<icatch filename="img/search-report" msg="タグ、提出者、テンプレート名、キーワードなど、様々な方法で日報や点検記録を検索・絞り込めます。目的のレポートを素早く見つけましょう" alice="here">}}

一部の検索条件は[無料プラン](/docs/price/free/)ではご利用できません。ご了承ください。

### 検索の条件を保存する - ワンクリックで検索を再実行{#saveQuery}

※この機能は[有料プラン限定](/docs/price/#fee)です。

頻繁に使用する検索条件は保存することで、次回以降ワンクリックで同じ検索を再実行できます。<strong>毎日確認する特定の日報</strong>や、<strong>毎週チェックする特定の設備の点検記録</strong>など、定型的な検索に非常に便利です。

保存したい検索条件で検索を行ったあと、その画面で「<strong>検索条件保存</strong>」ボタンをクリックします。

{{<icatch filename="img/save-query" msg="よく使う検索条件は保存すれば、次回以降はワンクリックで検索できます。日々の確認業務を効率化しましょう" alice="ok">}}

保存した検索条件は受信BOXの画面上部にボタン形式で表示されます。
検索の条件は複数保存できます。

検索条件の保存は基本的に絶対値ですが、検索期間については「今日・昨日・先週」といった<strong>相対的表現で保存した場合、実行日が変わると検索される条件範囲も相対的に変化する</strong>ことに注意してください。

### 検索条件の削除と検索のクリア{#clearQuery}

保存した検索条件を削除するには、削除したい検索ボタンの▼をクリックして削除します。
また、今の検索条件をすべて解除して通常の「高速モード」に戻るには「<strong>クリア</strong>」ボタンをクリックします。

{{<icatch filename="img/clear-query" msg="不要になった保存検索は削除しましょう。全ての検索条件を解除して高速モードに戻るには「クリア」ボタンをクリックします" >}}

### 表示対象と表示形式について{#viewMode}

表示対象と表示形式は画面でいうと以下の位置に配置されています。画像を参照してください。

{{<icatch filename="img/view-mode" msg="スマートフォンなら「カード」、PCなら「表」が見やすいでしょう。使いやすい表示形式を選んで、日報や点検記録を確認しましょう" alice="please">}}

表示対象は「<strong>受信BOX</strong>」「<strong>送信BOX</strong>」「<strong>下書きBOX</strong>」があります。

<dl class="basic">
<dt><strong>受信BOX</strong></dt>
<dd>あなたが<a href="/docs/manual/write-report/dist/">提出先または共有先</a>に指定されたレポート、つまりあなたが読めるレポートが表示されます。<strong>承認待ちの日報</strong>や、<strong>共有された点検記録</strong>などを確認できます。</dd>
<dt><strong>送信BOX</strong></dt>
<dd>あなたが作成し、提出したレポートが表示されます。<strong>過去の自分の報告内容</strong>や、<strong>提出済みの日報・点検記録</strong>を素早く確認できます。</dd>
<dt><strong>下書きBOX</strong></dt>
<dd><a href="/docs/manual/write-report/draft/">下書きとして保存したレポート</a>が表示されます。<strong>作成途中の日報や点検記録</strong>を編集・清書する際に利用します。</dd>
</dl>

表示されるレポートのタイプが変わるだけで、基本的な使い方はどのBOXも一緒です。

{{< callout context="caution" title="受信BOXが表示されないケース" icon="outline/alert-triangle" >}}
他人の日報を読む権限が無いため、[書き込みのみの権限](/docs/setup/staff-global/rank/#others)アカウントは受信BOXが表示されません。
{{< /callout >}}

表示形式は次の3種類があります。

<dl class="basic">
<dt><strong>カード表示</strong></dt>
<dd>スマートフォンにおすすめです。レポート1つ1つがカードで表示され、縦長のスマートフォン画面で見やすいレイアウトです。<strong>現場でのサッと確認</strong>に適しています。</dd>
<dt><strong>表表示</strong></dt>
<dd>PCにおすすめです。レポートが一覧で表示され、検索機能も組み込まれて使いやすいです。<strong>オフィスでの詳細なデータ確認や絞り込み</strong>に適しています。</dd>
<dt><strong>提出状況表示</strong></dt>
<dd>縦軸にスタッフ、横軸に日付の表で表示されます。<strong>未提出者や、特定の期間の提出状況を一目で把握</strong>するのに非常に便利です。<strong>日報の提出率チェック</strong>や、<strong>点検作業の進捗管理</strong>に活用できます。</dd>
</dl>

カードと表については特筆すべき点はありませんが、提出状況については次の章で補足説明をしています。

### 提出状況のレイアウト - 未提出者を一目で把握{#teishutuMap}

提出状況は、その名の通り「レポートの提出状況」を表にまとめたものです。
列に日付、行にスタッフを取り、交差したセルにはレポートの件数が表示されます。この数字はクリックすることで、該当レポートをポップアップで表示します。これにより、**誰が、いつ、どのくらいのレポートを提出したか**を視覚的に把握できます。

提出状況は、**検索や表示件数による影響を受ける**ことに注意してください。これはバグではなく意図的な動作です。
例えば検索で「承認済みの日報のみ」という絞り込みを行った状態で提出状況を表示すると、承認済みの日報のみで提出状況が表示され、未承認の日報は提出状況にそもそも反映されません。

この特性を活かして、<strong>承認された日報だけの提出状況を把握する</strong>など、目的に合わせた分析が可能です。ただし、意図せず検索した状態で提出状況を表示してしまうと、本来意図した結果とは異なる結果になる可能性があります。絞り込みがされていないことを確認するには「高速モード」であることを見れば一目瞭然です。

{{<icatch filename="img/report-map" msg="未提出者は誰ですか？提出状況表を使えば、日報や点検記録の提出状況を一目で把握し、必要な対応を素早く行えます" alice="ok">}}

{{< callout context="note" title="シフト管理と連動します" icon="outline/info-circle" >}}
提出状況表に欠勤やメモを書きたい場合は[シフト表](/docs/manual/utils/shift/)をご利用ください。提出状況表とシフト表は連動し、<strong>スタッフの勤務状況とレポート提出状況を合わせて確認</strong>できます。
{{< /callout >}}
