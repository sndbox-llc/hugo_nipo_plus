## スタッフのアカウントを新規作成する{#howto_make}

日報をチームで運用するには各スタッフの「アカウント」を作成し、適切に管理していく必要があります。  
アカウントの追加方法にはいくつかのやり方があります。お好みの方法でご利用ください。

<dl class="basic">
<dt><a href="#create_account">組織設定画面から一件アカウントを作成</a></dt>
<dd>最もシンプルな手順です。アカウント作成時にどのグループに所属させるかを選択ができます</dd>
<dt><a href="#batch_create">組織設定画面から一括でアカウントを作成</a></dt>
<dd>アカウントをまとめて登録する際に便利です。しかしパスワードや所属先など細かい設定は登録後に個別で行う必要があります</dd>
<dt><a href="#make_group">グループ設定画面から一件アカウントを作成</a></dt>
<dd>グループ設定内のスタッフ管理からも新規にアカウント作成が可能です。組織設定からの登録との違は、「どのグループに所属させるかの選択」が無く、作業しているグループに自動で所属する点です</dd>
</dl>

### 組織設定からスタッフアカウントを１件作成 {#create_account}

新しいアカウントを１件づつ作成する手順です。

1.  [左メニューから「組織設定」をクリック](/docs/setup/staff-global/rank/#rootSettingBtn)（管理者のみ表示）
2.  スタッフ管理セクションの中にある「**新規作成**」ボタンをクリック
3.  表示されたフォームに必要事項を入力し、「**作成**」ボタンをクリック

{{<icatch filename="img/create-new-staff" msg="組織内に新しいスタッフのアカウントを追加します。必須項目はE-mailアドレスのみです。" alice="guide">}}

#### 入力項目の解説{#input_fields_guide}

以下の表を参考に、各項目を入力してください。E-mailアドレス以外は任意入力です。

<dl class="basic">
<dt>ログインE-mail</dt>
<dd>スタッフがNipoPlusにログインする際に使用するメールアドレスです。ログインIDとしても使用します。</dd>
<dt>名前</dt>
<dd>NipoPlus内で表示されるスタッフの名前です。（例：山田 太郎）</dd>
<dt><a href="/docs/setup/staff-global/rank/">権限</a></dt>
<dd>このスタッフに与える権限を指定します。「管理者・編集者・スタッフ・書き込みのみ・ゲスト」の中から選択してください。</dd>
<dt>初期パスワード</dt>
<dd>ログイン時に使用する初期パスワードです。自動でセットされますが、「パスワードを変える」にチェックを入れることで、手動で変更することも可能です。</dd>
<dt>所属先</dt>
<dd>このスタッフが最初に所属するグループを指定します。初期状態ではすべてのグループに所属します。必要に応じて調整してください。</dd>
<dt>メモ</dt>
<dd>スタッフに関する任意のメモを記入できます。</dd>
</dl>

{{< callout context="tip" title="💡 ダミーのメールアドレスも使用可能" icon="outline/note" >}}
管理者以外のスタッフを追加する場合、**実在しないダミーのメールアドレス**を使用することも可能です。

これは、メールアドレスがパスワードリセットという重要な役割を担う一方で、**管理者が他のスタッフのパスワードをリセットできる権限**があるためです。また、従業員の数だけメールアドレスを用意できない会社も多いため、柔軟に対応できるようになっています。
{{< /callout >}}

### 組織設定から複数アカウントを一括作成する {#batch_create}

作成するユーザーアカウントが多い場合は、簡単に一括登録できる機能をご利用ください。

ログインに使用するE-mailアドレスのリストを用意し、入力エリアに1行につき1件のE-mailアドレスを貼り付けるだけです。最大50アカウントまでまとめて作成できます。

{{<icatch filename="img/webp" msg="複数のスタッフアカウントをまとめて作成できます" alice="pc">}}

{{< callout context="danger" title="【注意点】" icon="outline/info-triangle" >}}
一括作成の場合、登録される**権限**、**所属先グループ**、**初期ログインパスワード**は、すべて同じ設定となります。必要に応じて、アカウント作成後に個別に編集してください。
{{</callout>}}

### グループ設定からアカウントを１件作成{#make_group}

組織設定ではなく、作業中のグループ管理画面からスタッフアカウントを直接追加することもできます。  
内部的には「組織に追加」「当該作業グループへの所属」という手順が行われているため、追加の手順が異なっても結果は同一です。

{{<iTablet filename="img/makeUserFromGroup" msg="グループからも直接追加できるよー" alice="pc">}}

### 作成したアカウントでログインしてみよう {#login_account}

アカウントが作成され、グループに所属したら、そのアカウント情報（E-mailとパスワード）を該当のスタッフに伝えてください。
スタッフはNipoPlusを開き、その情報を使ってログインできるようになります。

ログインの具体的な方法については「[ログイン・ログアウトする](/docs/manual/account/signin/)」のページをご覧ください。

## アカウントの管理と運用 {#manage_account}

アカウントは作成後にも度々変更する必要が出てきます。例えばスタッフが休職や退職した際は当該アカウントを無効にする必要があります。  
異動や転勤により所属先のグループを変更する場合もあるでしょう。  
普段なかなか行わない操作になりますが、必要に応じて本ドキュメントをご参照ください。

### 所属するグループの変更 {#join_staff}

{{< callout context="note" title="グループを作成するには「[グループの作成](/docs/setup/make-group/)」をご覧ください" icon="outline/info-circle" />}}

アカウントはどこかのグループに所属して初めて作業が出来ます。アカウント作成時に所属グループを選択しますが、必要に応じてグループの脱退や再所属が行えます。

1.  左メニューから「**組織設定**」をクリック
2.  「スタッフ管理」セクションで、操作したいスタッフの行にある「**所属グループ**」をクリック
3.  グループ一覧が表示されるので、所属させたいグループにチェックを入れて「**保存**」をクリック

{{<icatch filename="img/join" msg="組織のスタッフをそれぞれのグループに所属させましょう。これにより、日報の読み書きが可能になります。" alice="book">}}

{{< callout context="note" title="グループから見たスタッフ一覧" icon="outline/info-circle" >}}
組織設定ではなく[グループ内のスタッフ管理画面](/docs/setup/staff-local/_about/)では、そのグループ内で固有の設定を行えます。この設定は他のグループには影響しません。
{{<icatch filename="img/group-member" msg="提出先の固定やテンプレートの制限などができます" alice="here">}}
{{< /callout >}}

### スタッフアカウントの基本情報を変更する {#change_staff_data}

スタッフの名前、権限、メモなど、アカウントの基本情報は、組織設定画面で直接クリックして簡単に変更できます。

1.  [左メニューから「**組織設定**」をクリック](/docs/setup/staff-global/rank/#rootSettingBtn)
2.  「スタッフ管理」セクションで、編集したいスタッフの該当項目（名前、権限など）をクリック
3.  表示される編集ウィンドウで、新しい値を入力・選択し、「**保存**」をクリック

{{<icatch filename="img/staff-edit" msg="スタッフの名前や権限、メモなどを変更するには、該当の項目を直接クリックしてください。ポップアップで編集画面が表示されます。" alice="here">}}

#### 変更可能な項目と注意点{#editable_fields_notes}

<dl class="basic">
<dt>名前・ふりがな</dt>
<dd>
    NipoPlus内で表示されるスタッフの名前です。「名前」をクリックして編集します。
    レポート作成時の提出者名や<a href="/docs/manual/read-report/state/#agree">承認ボタン押下時の名前</a>は、その時点で設定されている名前が使用されます。
</dd>
<dt>権限</dt>
<dd>
    スタッフの<a href="/docs/setup/staff-global/rank/">権限ランク</a>を変更します。
    <strong>自分の権限を変更することはできません</strong>。ご自身の権限を変更したい場合は、他の管理者権限を持つユーザーに依頼してください。
</dd>
<dt>所属グループ</dt>
<dd>
    スタッフが所属するグループを変更できます。
    <strong>どこのグループにも所属していないスタッフは、日報の読み書きなどNipoPlusの主要な機能を利用できません</strong>のでご注意ください。
</dd>
<dt>Email</dt>
<dd>
    ログインに使用するE-mailアドレスを変更できます。
    変更されたスタッフは、次回NipoPlusを起動する際に再ログインが必要になります。
    関連ページ：<a href="/docs/manual/account/email/">ログインE-mailの変更</a>
</dd>
<dt>メモ</dt>
<dd>
    スタッフに関する任意のメモを記入できます。クリックして編集してください。
</dd>
<dt>API利用</dt>
<dd>
    API機能の利用を許可するかどうかを設定します。API機能を使いたい場合は「⭕」にしてください。
    安全性を高めるため、API利用可能なアカウントは必要最小限に抑えることを推奨します。
    関連ページ：<a href="/docs/manual/api/ref/">APIを使う</a>
</dd>
<dt>パスワード</dt>
<dd>
    スタッフのログインパスワードを変更できます。
    変更されたスタッフは、次回NipoPlusを起動する際に再ログインが必要になります。
    関連ページ：<a href="/docs/manual/account/email/#password">ログインパスワードを忘れた方</a>
</dd>
</dl>

### アカウントの停止・削除（退職・休職時など） {#stop_remove}

スタッフの退職・休職などでNipoPlusを利用しなくなった場合、そのアカウントを適切に管理（無効化または削除）することで情報流出等のリスクを軽減できます。  
アカウントを無効にするには、「無効」と「削除」の２種類があります。通常は「無効」を選択してください。

#### スタッフのアカウントを無効化する（推奨） {#disable}

{{< callout context="note" title="無効化されたスタッフは、利用人数にはカウントされません" icon="outline/info-circle" />}}

アカウントの無効化は、アクセスを停止しつつ過去のデータとの紐付けを維持できるため、**最も推奨される方法**です。無効状態のアカウントは人数のカウントから除外され、当該アカウントはログインを含む一切の操作ができなくなります。  
すでにログイン中の場合、一定時間経過後（短時間トークンの失効後）ログインセクションが自動で解除されます。

1.  [左メニューから「**組織設定**」をクリック](/docs/setup/staff-global/rank/#rootSettingBtn)
2.  「スタッフ管理」の項目までスクロール
3.  無効化したいスタッフの「状態」列にあるボタンをクリック
4.  ボタンが赤色（または「無効」を示す表示）になれば無効化完了

{{<icatch filename="img/disable-account" msg="状態ボタンを1回クリックすると無効化、もう1回クリックすると有効に戻ります。" alice="ok">}}

{{< callout context="note" title="アカウントを有効に戻す場合はもう一度ボタンを押せばOK" icon="outline/info-circle">}}
しかし再ログインが必要になる可能性もありますのでその点ご留意ください
{{</callout>}}

### スタッフのアカウントを削除する {#remove}

アカウントの削除は、無効化よりもさらに強力な操作です。過去のレポートとの紐付けに影響が出る可能性があるため、**通常は「無効化」で十分**であり、削除は慎重に行うことを推奨します。

{{< callout context="caution" title="完全削除は退職後数ヶ月経ってからを推奨" icon="outline/alert-triangle" >}}

削除する際は、退職から数ヶ月ほど経過し、集計などに影響がないことを確認してから行うことを推奨しています。
{{< /callout >}}

1.  無効化ボタンの隣にある下向き三角ボタンをクリック
2.  表示されるメニューから「**削除する**」ボタンをクリック

{{<icatch filename="img/remove-account" msg="多くの場合、スタッフを「削除」する必要はほとんどありません。「無効化」で十分管理できます。" alice="ok">}}

**アカウント削除後の挙動：**

- アカウントがシステムから完全に削除されます。
- そのメールアドレスは空きの状態になるため、再び同じメールアドレスでアカウントの作成ができます。
- **しかし、削除前と削除後では異なるスタッフとして認識されます（IDが再割り当てされるため）。**

### 退職した管理者のアカウントを引き継ぐ {#hikitugi}

一般的なスタッフが退職した場合はアカウントの無効化で対応できますが、**管理者のスタッフが退職する場合**は少し注意が必要です。
新しい管理者のアカウントを作成しても、これまでに提出されてきたレポートが権限の関係で読めないケースが出てくる可能性があります（NipoPlusの設定により異なります）。

{{< callout context="note" title="退職した管理者のアカウントをそのまま新しい担当者が利用するのが最もシンプルな解決策です" icon="outline/info-circle">}}
引き継ぎの際にユーザ名を変更すれば、検印欄の名前で引き継ぎ前・引き継ぎ後の承認者がわかるためご安心ください。
{{</callout>}}

なお、引き継ぎの際は以下の作業をお忘れなく行って下さい

- 引き継ぐ際は、**安全のためログインパスワードを必ず変更してください。**
- アカウントを引き継いだら、**NipoPlus内で表示される名前を新しい担当者の名前に変更してください。**

### ログインできない「共用スタッフアカウント」を作成する {#dummy_account}

このアカウントは、ログイン可能な通常アカウントでログインしているユーザーが、「**誰がレポートを作成したか**」を自己申告するために使用する、少し特殊な用途のアカウントです。

{{<icatch filename="img/anony" msg="ログインできない共用アカウントは、特定の共有環境でのみ使われる特殊な用途です。" alice="ok">}}

**共用スタッフアカウントの主な特性：**

- **ログイン不可能:** このアカウント自体でNipoPlusにログインすることはできません。
- **権限が「書き込みのみ」で固定:** レポート作成に特化しており、他の権限は割り当てられません。
- **単体では利用不可:** 必ず他のスタッフの[共用設定](/docs/setup/staff-local/share/)として設定する必要があります。

**利用方法：**
ログイン可能な通常アカウントでNipoPlusにログインされている状態で、[レポート作成画面](/docs/manual/write-report/write/)の提出者を選択する欄から、利用者の名前としてこの「共用スタッフアカウント」の名前を選択してレポートを作成します。

**【注意点】**
レポート作成者が自己申告で名前を選択するため、**レポート提出者のなりすましを防ぐことはできません。** レポート作成者が本人であることをシステムで保証する必要がある場合は、正規のログイン可能なアカウントの利用をご検討ください。

</details>
